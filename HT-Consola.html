Git por consola

Getting Started

Instalación

Para instalar git en nuestro equipo ejecutaremos en la consola el comando:

:$ sudo apt-get install git

Para comprobar que esta instalado y pasa saber la versión ejecutamos el comando:

:$ git --version

Git init

Primero abrimos el terminal y vamos a la carpeta donde esta el archivo que queremos seguir con git.

Una vez alli, para hacer el seguimiento con guit hacemos el comando:

:$ git init

Configurar Git

Para configurar y añadir nuestra cuenta, haremos los siguientes comandos:

:$ git config --global user.name "name"

:$ git config --global user.email "email"

Para consultar los valores de estas variables en el futuro, podemos usar el comando:

:$ git config --global --list

Git Clone

Crea una copia local del repositorio local ejecutando:

$: git clone /path/to/repository

Si queremos usar un servidor remoto, usaremos el comando:

$: git clone username@host:/path/to/repository

Guardar cambios

Git add

Para añadir el archivo hacemos el comando:

:$ git add "Archivo/os"

Como hacer un commit

para hacer un commit pondremos en consola el siguiente comando:

:$ git commit -m ”mensaje que queremos añadir”

Si solo ponemos :$ git commit nos saldra el editor predeterminado para añadir el mensaje del commit

Inspecting a repository

Git status

Este comando sirve para ver el estado de los arhivos.

Para ver el estado de nuestros archivos tendremos que hacer el comando:

:$ git status

Los arhivos pueden estar en tres estados diferentes:

  • Committed
  • Modified
  • StAged
  • Untracked

Aquí os dejo una imagen con el ciclo de vida del estado de los archivos

Git log

El comando git log sirve para ver los commits que hemos echo. Se puede ver con el comando:

$:git log --oneline --decorate

Los 40 caracteres al lado de la palabra commit son dos cosas:

  1. Una suma de verificación SHA-1
  2. El identificador unico del commit

Undoing changes

Git revert

Genera un nuevo commit que deshace todos los cambios introducidos en "commit", luego lo aplica a la rama actual

El comando para realizar un revert sería:

:$ git revert "commit"

Collaborating

Git remote

Este comando se utiliza para añadir una cuenta remota. (Por ejemplo una cuenta de git hub para subir nuestros proyectos).

Para crear una nueva conexión remota hacemos el comando:

:$ git remote add "nombre" "url"

Aquí os dejo un ejemplo

git remote add lito http://dev.example.com/lito.git

Git fetch

Este comando importa un repositorio remoto (normalmente de otra persona) en tu repositorio

Si queremos hacer un fetch usaremos el comando siguiente:

:$ git fetch "ruta remota"

Git pull

Se usa para actualizar tu repositorio local al commit más nuevo, para esto ejecutaremos:

$: git pull

En tu directorio de trabajo, esto baja y fusiona los cambios remotos.

Git push

Nuestros cambios están ahora en el HEAD de tu copia local. Para enviar estos cambios a tu repositorio remoto ejecutamos:

$: git push origin master

Tienes que remplazar master por la rama a la que quieres enviar tus cambios.

Si no has clonado un repositorio ya existente y quieres conectar tu repositorio local a un repositorio remoto, usa:

$: git remote add origin "server"

Ahora podrás subir tus cambios al repositorio remoto seleccionado.

Tambíen esta la opción de subir una rama, para ello ejecutaremos el comando:

$: git push origin "Nombre de la rama"

Ramas

Como hacer una rama nueva

Para crear la rama y cambiarnos a ella, usamos el comando:

$: git checkout -b "nombre de la rama"

Si solo queremos cambiar de rama, volviendo por ejemplo a la rama principal, utilizaremos el comando:

$: git checkout master

Como eliminar una rama

Pongamonos en el caso que queremos borrar la rama "lito", para esto, tenemos que hacer el comando:

:$ git branch -d lito

Como cambiar de ramas

Para cambiar de la rama actual a la rama Juan, tenemos que hacer el comando:

$: checkout Juan

Git merge

Este comando se utiliza para fusionar otra rama a tu rama activa (por ejemplo master).Para ello utilizaremos el comando:

:$ git merge "y aqui pondremos la rama"
